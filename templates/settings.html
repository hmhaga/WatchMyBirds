Auto follows the active model: —. Concretely it computes model_floor + 0.10, so a YOLOX-Tiny with a floor of 0.15 yields an effective threshold of 0.25 — relatively permissive. The Manual fallback default is 0.65. Manual uses whatever value you set here (typical tightening: 0.80–0.90). The threshold is applied per detection: in a frame with multiple birds, only those scoring at or above the threshold are persisted — weaker companions are dropped. Higher values save disk space and review time at the cost of dropping borderline detections. Changes take effect on the next detection cycle (no restart needed).
marten · cat · squirrel · hedgehog
Bird detections ride a CLS sanity check, non-bird OD classes do not — so they need a stricter floor. Default 0.80 is calibrated against live RPi data: marten_mustelid peaks at OD-conf 0.77 on static-bbox night triggers, while a genuine cat near the feeder reaches 0.92. Below this floor the detection is gated to UNCERTAIN and hidden from Stream / Gallery / Telegram. Bird detections (Tauben, Eichelhäher, Blaumeise, …) are not touched by this knob regardless of their OD confidence.
On: non-bird detections below the floor are dropped before the database write — no image saved, no compute spent. Saves ~250 MB per night against static-bbox triggers like the marten background false positives. Off: keep them as UNCERTAIN rows in the DB (still hidden from Stream / Gallery / Telegram). Useful during a Phase-7 bbox-cluster collection window when you want to see where the FPs cluster on the camera.

Sliding-window cap that protects the review queue from flocks of common species. Default 100 per 60s (set to 0 to disable). The Save Threshold above runs first, so weak detections never consume burst slots. Skipped bursts are noted in the log every 30s.
Tuning rule of thumb: if the review queue still overflows, halve this cap first (50), then extend the window below (120s) to throttle the rate further. Only after that, raise the Save Threshold.
seconds
Window length for the Burst Cap. Default 60 seconds — i.e. with the default cap of 100, at most 100 detections will be saved per minute, regardless of how many the detector returns. Lengthen this (e.g. 120s) to throttle the saved rate further without lowering the cap.

WatchMyBirds runs entirely on your Pi. Nothing about your bird activity, images, or detections ever leaves the device.

Optionally, you can let this installation send one anonymous heartbeat per day so we can count how many people actually use the project. The heartbeat contains a random installation ID, the app version, and basic hardware info (CPU count, rounded RAM). No images, no detections, no IP, no location, no email.

See Privacy page for the full list of what we collect and what we explicitly never collect.
